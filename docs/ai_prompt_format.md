# Chartici AI Generation — Step 1 (Common)

Общая часть формата AI-генерации, одинаковая для всех типов диаграмм.  
Шаг 2 (тип-специфичный Markdown вывод) описан в `src/engines/<type>/ai_prompt.md`.

---

## Архитектура генерации

```
Запрос пользователя
  ↓
[Шаг 1] LLM: <thinking>...</thinking>
         Рассуждение, план диаграммы, выбор групп и нод.
         Не парсится — только для отладки.
  ↓
[Шаг 2] LLM: Markdown-таблицы (зависит от engine)
         Парсер: src/services/aiGenerate.js
  ↓
внутренний граф {nodes, edges, groups}
```

## Шаг 1: блок `<thinking>`

LLM начинает ответ с:

```
<thinking>
...размышления о структуре диаграммы...
</thinking>
```

- Блок закрывается `</thinking>` до начала Markdown-таблиц.
- Содержимое **не парсится** — только для отладки и логирования.
- Язык: произвольный (LLM выбирает сам под запрос).

## Общие правила шага 2

- Далее идут секции с `#` заголовками и `###` подсекциями.
- Каждая подсекция содержит Markdown-таблицу нод.
- Заголовок подсекции кодирует метаданные группы (имя, размер, цвет, родитель).
- **Язык меток:** LLM сохраняет язык запроса (русский запрос → русские метки).

## Размеры и семантика

Каждый engine определяет семантическую шкалу в `engine.js → ai_prompt.semanticScale`.  
LLM видит словесные обозначения (например, `"process"`), парсер маппит обратно в `S/M/L`.

*Тип-специфичные форматы: `src/engines/<type>/ai_prompt.md`*
